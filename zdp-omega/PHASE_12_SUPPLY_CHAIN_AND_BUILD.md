# PHASE 12 — SUPPLY CHAIN & BUILD INTEGRITY
> Most modern code is other people's code. You either know exactly what's in your artifact, where it came from, and that it was built untampered — or you're shipping unknowable risk.

**Lead standards:**
- **SLSA v1.x** (Supply-chain Levels for Software Artifacts) — provenance levels L0–L3.
- **SPDX (ISO/IEC 5962)** & **CycloneDX** — SBOM formats.
- **NIST SP 800-218 (SSDF)** & **800-161** (supply chain risk).
- **ISO/IEC 5230 (OpenChain)** — license compliance.
- **CIS Software Supply Chain Security Guide**; EO 14028 SBOM expectations.
- **Sigstore / in-toto** — signing & attestation practice.

**SIL depth:** SIL3/4 artifacts require signed provenance + reproducibility evidence. **Inputs:** dependency inventory (P01), Phase 10 secrets scan.

## 12.1 MISSION
Every runtime artifact ships with a signed SBOM + provenance attestation. Every dependency is pinned, verified, license-cleared, and vulnerability-scanned. The build pipeline itself is a hardened, auditable factory.

## 12.2 FALSIFICATION PROCEDURES — RUN AS THE RIVAL

> **Mind: THE RIVAL.** Hostile, no benefit of the doubt, no courtesy notice. Full stance: `APPENDIX_RIVAL_AND_PRINCIPAL_DOCTRINE.md` §E.1.

- **Reproducibility probe:** rebuild the artifact from the same commit twice; hashes differ = non-reproducible (record baseline, aim higher).
- **Typosquat / confusion probe:** verify each package name against the intended source (package registry + org); look for lookalikes, dependency-confusion vectors (internal names publishable on public registries), and namespace hijacks.
- **Lockfile-lie probe:** temper (in a test branch) with a locked hash; the build must refuse. If it silently upgrades, integrity is theater.
- **Post-install probe:** review install/build/postinstall scripts of top-N dependencies for network calls, environment reads, or filesystem writes outside the build sandbox.
- **License-shock probe:** for each dependency license, ask "does this restrict our distribution/use per TIMELINE?" Copyleft in a proprietary distribution path = P0/P1.

## 12.3 EXHAUSTIVE RULES

### Dependencies
- **R12.1** All direct + transitive dependencies pinned by **exact version + integrity hash** in a committed lockfile; the build **verifies hashes** and fails on mismatch.
- **R12.2** Dependency-confusion prevented: private packages scoped/namespaced; registry configured so internal names cannot be shadowed by public ones; source registry pinned per dependency.
- **R12.3** **Vulnerability scanning** on every push (SCA) against known-vuln DBs (NVD/OSV/GHSA); policy: **no known-exploited-vulnerability (KEV) in production**; time-to-remediate SLAs by severity (documented in TIMELINE NFR).
- **R12.4** Unmaintained/abandoned packages flagged (no releases > policy window, single-maintainer risk); replacement plan in DECISIONS.md when found.
- **R12.5** Update cadence defined (e.g., automated PRs, security patches within SLA); auto-merge only for security patches passing gates.

### SBOM & provenance (SLSA)
- **R12.6** **SBOM produced per build** in **SPDX or CycloneDX**, published as a build artifact, versioned with the release. Includes components, versions, licenses, cryptographic hashes, and relationships.
- **R12.7** **Provenance attestation** (in-toto / SLSA) generated by the build system, signed (Sigstore-class), and verifiable by consumers. Target: SLSA **L3** for production releases (hosted, hardened, non-falsifiable provenance); SIL2/1 modules may accept L2.
- **R12.8** Artifacts are **signed**; deployment verifies signatures before running; unsigned or mismatched artifacts refused.
- **R12.9** **Reproducible builds** targeted: deterministic toolchain versions, no build-time timestamps in artifacts, sorted inputs. Divergences documented.

### Build pipeline hardening
- **R12.10** Build runs on **ephemeral, hermetic** runners; no long-lived shared build hosts; no network fetches outside a controlled proxy/mirror.
- **R12.11** Least-privilege credentials: short-lived OIDC-federated tokens over static keys; secrets scoped per job; publishing tokens (npm/pypi/etc.) tightly scoped and rotated.
- **R12.12** **Two-person rule** for release-signing keys / production publish; branch protection with required reviews on main/release branches (ties P23).
- **R12.13** Post-install/preinstall/build scripts of dependencies sandbox-executed or disabled where feasible; hooks reviewed on version bumps.
- **R12.14** Base images / container/OS layers pinned by digest (never floating tags); minimal base; regular rebase; user is non-root; read-only rootfs where possible.

### License & IP compliance (OpenChain)
- **R12.15** License inventory attached to SBOM; each license classified (permissive / weak copyleft / strong copyleft / proprietary / unknown); each classification checked against the distribution model in TIMELINE.
- **R12.16** Attribution/NOTICE file generated automatically from SBOM for shipped products.
- **R12.17** Unknown/ambiguous licenses treated as blocking until resolved.

### Trust of the tools
- **R12.18** The tools that build/lint/scan/deploy are themselves inventoried and version-pinned (turtles all the way down); tampering with the pipeline is treated as a P0 security incident (feeds P17 alerting).
- **R12.19** Release process end-to-end auditable: source commit → build → SBOM → provenance → sign → publish → deploy is fully logged and cross-referenced by a release ID.

## 12.4 EXIT GATE — JUDGED AS THE PRINCIPAL

> **Mind: THE PRINCIPAL.** Guilty until the evidence tier matches; no gate closes on "mostly." Full stance: `APPENDIX_RIVAL_AND_PRINCIPAL_DOCTRINE.md` §E.2.

- [ ] Lockfile integrity enforced; tamper test fails the build. **Evidence:** test transcript.
- [ ] SBOM generated per build (SPDX/CycloneDX); attached to release. **Evidence:** SBOM sample + pipeline step.
- [ ] Provenance attestation signed & verifiable; SLSA target level met for production. **Evidence:** attestation sample + verify log.
- [ ] Vulnerability scan clean of policy-violating findings; SLAs defined and tracked. **Evidence:** scan report + SLA doc.
- [ ] Base images / OS layers pinned by digest; non-root runtime. **Evidence:** config.
- [ ] License inventory clean vs distribution model; NOTICE file present. **Evidence:** inventory + NOTICE.
- [ ] Hermetic build runners + short-lived credentials confirmed. **Evidence:** pipeline config.
- [ ] Reproducibility status recorded (baseline hash comparison). **Evidence:** rebuild report.

## 12.5 ARTIFACTS OUT
Committed lockfiles; SBOM(s); provenance attestations; vuln-scan reports; license inventory + NOTICE; hardened pipeline definitions; release audit trail.
